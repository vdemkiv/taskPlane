# Localization / i18n (optional) lens

**Group:** Experience
**Charter:** works across languages and locales
**Does NOT own:** content accuracy → tech-writer

## Looks for
externalized strings, locale formatting, pluralization, RTL, timezone/currency

## Fires when
- files match: **/locales/**, **/i18n/**, **/*.po, **/*.pot, **/lang/**, **/translations/**
- task types: ui

## Evaluator prompt

You are reviewing this change through the **Localization / i18n (optional)** lens only. Your charter: works across languages and locales. Stay inside it — anything under “content accuracy → tech-writer” belongs to that lens; note it in one line and move on.

Examine, with file:line evidence:

1. User-visible strings externalized to the locale system — none inlined.
2. No sentence-building by concatenation; templates with named placeholders.
3. Dates, numbers, currency formatted per locale, not hardcoded formats.
4. Pluralization via the locale rules (not `count > 1`).
5. Layout survives RTL and 2× string length.
6. Timezones explicit: stored UTC, displayed local.

**Blocker** = a user-facing flow that cannot be localized (hardcoded strings in core UI).
**Major** = concatenated sentences; `n > 1` pluralization; TZ-naive datetimes.
Minor = worth fixing, doesn't gate. Prefer the smallest suggestion that resolves each finding.

## How this lens runs

- **Prime (EXECUTE/FIX):** the loop hands the executor this lens's charter +
  looks-for BEFORE building — build so the review below finds nothing.
- **Review (EVALUATE/EM):** apply the evaluator prompt to the diff. `inline`
  mode: the evaluator applies it directly. `subagent` mode: it runs as its own
  read-only governed agent and returns the verdict JSON.

## Verdict format (all lenses)

Return findings, then a verdict. A finding without file:line evidence is an
opinion — mark it `question`, not `blocker`.

```json
{"lens": "<id>",
 "findings": [{"severity": "blocker|major|minor|question|praise",
               "file": "path", "line": 0,
               "issue": "what is wrong", "why": "the principle",
               "suggestion": "smallest fix that resolves it"}],
 "verdict": "pass|fail",
 "confidence": "high|medium|low"}
```

`fail` only when at least one **blocker** stands. Majors don't fail the gate
alone but must be listed for the EM synthesis and the fix cycle.
